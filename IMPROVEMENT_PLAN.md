# Improvement plan: Local Booking TMA

Основано на `CUSTOMER_REVIEW.md`. Цель - превратить технический booking-demo в Mini App, который выглядит как реальная запись в конкретную студию.

## Целевое направление

Ниша: beauty studio или private grooming studio.  
Визуальный стиль: warm ivory, graphite, soft rose, живые фото студии и мастеров.  
Главное ощущение: пользователь уверен, куда он записывается, к кому, на какое время и что будет после подтверждения.

## Фаза 1. Дизайн-система

- Заменить универсальный зеленый акцент на отдельную палитру студии.
- Добавить брендовые переменные: `--brand-bg`, `--brand-ink`, `--brand-accent`, `--brand-soft`, `--brand-danger`.
- Пересобрать первый экран как профиль бизнеса: обложка, адрес, рейтинг, ближайший свободный слот, основной CTA.
- Убрать ощущение набора одинаковых карточек: услуги оформить как прайс-лист, мастеров как профили.
- Сделать разные состояния слота: free, selected, busy, blocked, pending, confirmed.
- Сделать desktop-preview как презентационный frame: телефон в контексте, рядом краткий business summary.

## Фаза 2. Функционал

- Добавить перенос записи: выбор новой даты/слота, подтверждение изменения.
- Добавить waitlist для занятого слота.
- Добавить правила отмены и предупреждение перед отменой.
- Добавить mock-подтверждение телефона кодом.
- Добавить комментарий к записи с валидацией.
- Добавить конфликтное состояние: слот стал занят во время подтверждения.
- Улучшить админку: календарь по мастерам, блокировка слота, список записей по дню.

## GSAP-анимации

Добавить зависимости:

```bash
npm install gsap @gsap/react
```

Где применить:

- Entry reveal первого экрана: обложка студии мягко масштабируется с `scale: 1.04 -> 1`, блок рейтинга и ближайшего слота появляются stagger-анимацией.
- Booking timeline: при переходе `услуга -> мастер -> дата -> время -> подтверждение` активный шаг плавно сдвигается, а старый уходит через `opacity` и `y`.
- Slot selection: выбранный слот получает короткий spring-like pulse через `scale`, busy-slot не анимировать, чтобы не путать пользователя.
- Confirmation moment: после успешной записи показать booking pass, который выезжает снизу и слегка "садится" на место.
- Admin calendar: при смене дня записи появляются stagger-списком, без резких скачков.

Ограничения:

- Все анимации отключать при `prefers-reduced-motion: reduce`.
- Не анимировать цену, дату и время так, чтобы пользователь мог неверно прочитать данные.
- Не использовать бесконечные декоративные циклы.

## Технические шаги

- Создать `src/motion/useReducedMotion.ts`.
- Создать `src/motion/bookingMotion.ts` с функциями для timeline, slots, confirmation.
- Разбить крупный `App.tsx` на компоненты: `BusinessHeader`, `BookingTimeline`, `ServiceList`, `SpecialistPicker`, `SlotGrid`, `BookingPass`, `AdminCalendar`.
- Добавить loading, empty и error states для mock API.
- Проверить `npm run lint`, `npm run build`, mobile 390x844 и desktop 1280x720.

## Критерии готовности

- Первый экран сразу показывает конкретную студию, а не универсальный booking-template.
- Пользователь видит главный следующий шаг без чтения инструкции.
- Запись, перенос и waitlist проходят без визуальных скачков.
- Слоты и статусы нельзя перепутать.
- GSAP делает переходы понятнее, а не просто украшает экран.
